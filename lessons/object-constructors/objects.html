Prototype Inheritance

Since we have the idea to reuse the function without repetitive declaration of object. There is a great way also that we can clone the object with declaring the same properties. It is called Prototype Inheritance.

There are three ways to implement the Prototype Inheritance:

  1. Using Object.setPrototypeOf
  2. Using Object.create "Constructor Method"
  3. Using ES6 Class (Modern JavaScript Implementation)

1. Using Object.setPrototypeOf()

It shows how to inherit the constructor by using Object.setPrototypeOf.

  1. Parent Construtor is the Book function with the properties of title, author, pages, and read. Instead of including the function/method info() inside the Parent Constructor, I separate it to make it readable and doing the standard for creating function for inheritance.

  2. Child Constructor is the SelfImprovementBook function. To inherit the class from Parent Constructor (Book Function), it must use ParentConstructor.call(this, ...) to inherit and initialize shared properties. Also to add a new properties, you must put after the ParentConstructor.call() method.

  3. In using Object.setPrototypeOf(param1,param2), it has two parameter.
    Param1 = Child Constructor (SelfImprovementBook)
    Param2 = Parent Constructor (Book)
    In this way, we inherit the properties from Parent to Child Constructor.

  4. I add function/method named showTopic to do some function. In order to do that. I must call ChildConstructor.prototype.showTopic. Just like in the previous way of creating methods for cleaner approach.

  5. In order to use the inherited prototype, I must create new instances which is book1 and book2 and using new keyword to use created constructor.

2. Using Object.create() "Constructor Method"

It shows how to inherit the constructor by using Object.create(). It is the same as Object.setPrototypeOf() but it differs usage scenarios.

  1. Parent Construtor is the Book function with the properties of title, author, pages, and read. Instead of including the function/method info() inside the Parent Constructor, I separate it to make it readable and doing the standard for creating function for inheritance.

  2. Child Constructor is the SelfImprovementBook function. To inherit the class from Parent Constructor (Book Function), it must use ParentConstructor.call(this, ...) to inherit and initialize shared properties. Also to add a new properties, you must put after the ParentConstructor.call() method.

  3. In using Object.create(param1), it has one parameter.
    Param1 = Parent Constructor (Book)
    In order to use it. We have a pattern to use:
    ChildConstructor.prototype = Object.create(ParentConstrutor.prototype);
    In this way, we inherit the properties from Parent to Child Constructor.

  4. This will be different from Object.setprototypeof because we need to point our inherited constructor which is Child Constructor to itself to work properly. To do it, we must have a pattern that
    ChildConstructor.prototype.constructor = ChildConstructor;

  5. I add function/method named showTopic to do some function. In order to do that. I must call ChildConstructor.prototype.showTopic. Just like in the previous way of creating methods for cleaner approach.

  6. In order to use the inherited prototype, I must create new instances which is book1 and book2 and using new keyword to use created constructor.

What is the difference between Object.create() and Object.setPrototypeOf()?

Feature Object.setPrototypeOf() Object.create()
When to Use When you already have a prototype and want to modify it. When creating a new prototype object from scratch.
Modification Modifies an existing prototype dynamically. Creates a new prototype linked to the given object.
Performance Slower (dynamic changes force engine de-optimization). Faster (prototypes set during object creation).
Constructor Reference No need to reset constructor reference. Must reset constructor manually.
Use Case Useful for retrofit inheritance or runtime changes. Better for initial inheritance setup.

Which One to Use?

  • Object.create() is preferred when you want to set up inheritance from the start. It creates a new prototype object, making it clearer and more efficient for initial setups.
  • Object.setPrototypeOf() is useful if you need to modify inheritance dynamically, such as when working with existing objects or prototypes that were not initially linked correctly.

3. Using ES6 Class (Modern JavaScript Implementation)

It is different to the two examples above and this is the standard and modern way to inherit prototypes.

  1. Parent Class is the Book function with the properties of title, author, pages, and read. I include the function/method info() inside the Parent Class since this is the way to implement it.

  2. Child Class was created and automatically insert word "extends" after the naming of child class and will be followed by parent class to inherit that properties and methods from parent class.

  3. Inside the child constructor I use super() method instead of call() because this is the standard for the class inheritance. The implementation of super is same with call but without using "this" keyword. Just insert the properties from parent and same implementation when adding new properties from created child constructor.

  4. I created showTopic() method inside child class since it will be separated function exclusive this child class only.

  5. In order to use the inherited prototype, I must create new instances which is book1 and book2 and using new keyword to use created constructor.

Advantages of ES6 Classes

  • Cleaner syntax: Classes provide a more readable and concise way to define inheritance.
  • super() keyword: Simplifies calling the parent constructor and methods.
  • Encapsulation: Methods are defined directly within the class, keeping code organized.
  • Prototypical inheritance under the hood: Even though classes offer syntactic sugar, they still follow JavaScriptâ€™s prototype-based inheritance model.
